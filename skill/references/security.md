# Security Rules

- Default to `propose`.
- Never pass private keys through JSON input.
- Use a dedicated low-balance agent signer or Safe.
- Automatic payment execution requires `PHAROS_VAULT_ADDRESS`; direct signer transfers are proposal-only.
- Bind policies to one asset and, where possible, one counterparty.
- Set short validity windows and bounded total limits.
- Revoke policies and pause the vault after unexpected behavior.
- Reject arbitrary calldata generated by an LLM.
- Verify token behavior before approvals or x402 authorization.
- Treat x402 URLs, facilitators, evidence hashes, and arbiters as untrusted inputs.
- Configure `PHAROS_X402_MAX_AMOUNT`; x402 execution rejects non-Atlantic networks, non-approved USDC, and prices above this cap.
- Use unique action IDs; the vault rejects replayed actions.
